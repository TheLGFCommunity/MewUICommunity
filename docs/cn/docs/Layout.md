# 布局（测量 / 排列 / 渲染）与 DPI 像素对齐

本文档介绍 MewUI 使用的布局/渲染协定，以及如何在不同 DPI 下保持布局稳定并避免出现 1 像素的裁剪瑕疵。

## 术语

- **DIP**：与设备无关的像素（逻辑单元）。在 MewUI 中，大多数布局坐标/尺寸都以 DIP 表示。
- **Px**：设备像素（物理像素）。`px = dip * dpiScale`。
- **dpiScale**：`Dpi / 96.0`。
- **约束**：传递给 `Measure(...)` 的 `Size availableSize`。
- **期望尺寸**：元素在 `Measure` 之后的首选大小。
- **边界**：元素在 `Arrange` 之后被安排的矩形区域。

## 流程概览

MewUI 采用即时模式 UI 的方式（每帧绘制），但会保留布局树用于命中测试以及在帧之间复用布局结果。

流程如下：

1) **测量**：自顶向下，确定 `DesiredSize`。
2) **排列**：自顶向下，分配 `Bounds`。
3) **渲染**：使用 `Bounds` 和当前视觉状态进行绘制。

### 各阶段的执行时机

- `InvalidateMeasure()` 会将该元素（及其上级）标记为需要新的测量阶段。这也意味着排列已失效。
- `InvalidateArrange()` 会将该元素（及其上级）标记为需要新的排列阶段，但不一定需要重新测量。
- `InvalidateVisual()` 会请求重绘，但不应该改变布局。

**经验法则**：
- **滚动** 应仅涉及 *排列 + 渲染*（偏移量变化通常需要重新排列子元素，而不是重新测量）。
- **影响大小/内容的属性更改** 需要 **测量**。

## 测量

### 目的

测量用于计算元素在给定约束下 *期望* 占据的大小。

### 输入 / 输出

- 输入：以 **DIP** 为单位的 `availableSize`
- 输出：以 **DIP** 为单位的 `DesiredSize`

### 规则

- 测量必须相对于布局状态是 **纯** 的：
  - 它不能在未先检查变化的情况下修改影响布局的属性（以避免无限失效循环）。
  - 它不应该依赖于先前安排的边界。
- 如果满足以下条件，可以跳过重新测量：
  - 该元素不是测量脏的，并且
  - 约束条件没有改变。

### 测量中的 DPI 舍入

测量返回的是 DIP 大小。布局系统可能会应用 *布局舍入*，以使大小在不同 DPI 下保持稳定，并避免产生导致 1 像素缝隙的分数边缘。

重要原则是：
- 测量不应尝试使用父坐标来“手动进行像素对齐”。
- 推荐在布局系统的边界处应用一致的舍入策略（例如在分配 `DesiredSize` 时）。

## 排列

### 目的

排列为每个元素分配最终的位置和大小（`Bounds`）。

### 输入 / 输出

- 输入：以 **DIP** 为单位的 `finalRect`
- 输出：以 **DIP** 为单位的 `Bounds`

### 规则

- 如果满足以下条件，可以跳过排列：
  - 该元素不是排列脏的，并且
  - 安排的边界与之前相同。
- 排列负责放置子元素（例如内容演示器、面板）。

### 排列中的 DPI 舍入（像素对齐）

视觉瑕疵（细线、1 像素裁剪）的主要来源是以下各方之间的舍入不一致：
- 父级计算出的子矩形，
- 子级自身的舍入，
- 渲染时的裁剪。

为避免这种情况：

- **在整个阶段使用单一的 dpiScale**（窗口 DPI）。
- **对齐矩形的边缘**，而不仅仅是宽度/高度：
  - 同时对左右边缘进行舍入可能会在向内舍入时使大小缩小 1 像素，
  - 因此，需根据语义有选择地使用向内或向外舍入。

推荐的语义：

- **用于绘制边框/背景的边界**：对齐 *边缘*，使笔划落在设备像素上。
- **视口 / 裁剪矩形**：**向外** 对齐，这样裁剪区域不会因为舍入而变得比预期的视口更小。

## 渲染

### 目的

渲染使用 `Bounds` 和当前视觉状态来绘制元素。

### 规则

- 渲染不能执行布局操作。在渲染内部不能调用测量/排列方法。
- 应尽可能使用已经对齐的几何图形进行绘制。

### 裁剪规则（1 像素瑕疵）

文本、笔划和抗锯齿可能会延伸到逻辑边界外半个像素。如果上级元素恰好在其子元素边界处应用裁剪，这些超出部分可能会被裁剪掉，看起来像是“右侧/底部缺少了 1 像素”。

推荐的方法：

- 视口内部的内容渲染应：
  1) 以 DIP 为单位计算视口矩形，
  2) 将其对齐到像素（优先使用向外舍入），
  3) 应用裁剪，
  4) 在绘制边缘的 1 像素笔划/字形时，可以选择在右侧/底部将裁剪区域扩展 **+1 设备像素**（如果你知道内容可能会超出）。

这就是类似“按设备像素扩展裁剪区域”这类辅助方法的设计意图。

## DPI 传播与缓存

### DPI 可信源

- 窗口提供有效的 DPI（`Dpi`、`DpiScale`）。
- 子级在测量/排列/渲染中做舍入决策时，应使用相同的 DPI。

### 缓存

依赖于 DPI 的缓存（文本测量缓存、几何图形缓存等）必须在以下情况发生时失效：
- 元素的有效 DPI 发生变化，
- 与字体相关的属性发生变化，
- 文本的换行/约束发生变化。

## 滚动：布局期望

### 偏移量

- 概念上以 DIP 为单位存储偏移量，但为了稳定的像素对齐，你可以在内部保留以像素为单位的值。
- 除非内容范围/视口发生变化，否则更改偏移量不应要求重新测量。

### 滚动期间应更新的内容

1) 使用 `viewport - offset` 排列子元素。
2) 在视口裁剪下渲染内容。
3) 更新滚动条的范围/值，使其与当前偏移量和视口匹配。

## 反模式（会导致性能问题或无限失效）

- 在测量/排列期间，未先比较新旧值就设置影响布局的属性。
- 从渲染方法中触发 `InvalidateMeasure()`。
- 在热路径中未加缓存地遍历树来重新计算 DPI 缩放比例。
- 在每次滚动时（仅偏移量发生变化）都进行测量。
